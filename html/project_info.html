项目介绍

环境介绍

Library-Manage-System由LAMP开发环境搭配HTML, JavaScript, CSS实现。

Linux Apache2 Mysql PHP
Ubuntu-20.02-WSL Apache/2.4.41 (Ubuntu) 8.0.36-0ubuntu0.20.04.1 (Ubuntu) PHP Version 7.4.3-4ubuntu2.22

Mysql数据库设计使用了MysqlWorkBench8.0CE。

JavaScript使用过程中应用的第三方库: jquery-3.7.1.min.js, jsencrypt.min.js。

项目功能

Library-Manage-System是一个完全无法配置，维护难度较大，功能增加修改难度较大的屎山项目。

  1. LMS实现了用户登录功能，包含5不同类型的用户权限访问控制:
    • 游客
    • 普通用户
    • 管理员
    • 中级管理员
    • 超级管理员 其中游客身份只有在特殊操作或者session超时才会生效。
  2. 用户口令哈希存储，LMS在用户隐私信息传输过程中采用了RSA双端加密传输，使用公钥在前端加密，私钥写在私有路径下，使用后端php代码读取后用于解密。同时用户的密码信息将会在私钥解密后使用password_hash函数加密后写入数据库中，使用password_verify函数校验密码。
  3. 体现数据库完整性检查，设置了主键、外键等，保证数据库完整性。
  4. 体现防 SQL 注入。用户输入表单后通过前后端双端校验，避免SQL注入。
  5. 具有数据备份与恢复功能。权限大于等于3的用户可以进行数据库备份与恢复操作。

页面外观

LMS有着相对简洁甚至朴素的管理界面，能够面对不同权限的用户动态渲染导航栏入口，同时实现了双端鉴权。

LMS的主要功能分为6大板块：首页、书库、用户列表、系统管理、个人中心、关于。你可以在左侧的竖向导航栏切换这6大板块。

其中每一大板块可能有着多个小板块，你可以在上方的横向导航栏切换小板块。

不同大板块中的每一个小板块都有着对应的最低权限，如果登录用户的权限不满足该最低权限。则该按钮将不会在导航栏上显示。

横竖导航栏均有一个slider控件，每一次板块的切换都会从上一个板块移动至当前的板块，用于高亮显示当前所处位置的同时实现页面加载的动态过渡。

项目中包含一个Loading控件、每一次小板块的切换都会短暂显示该控件，直到页面加载完毕。

项目结构

前端

系统前端主要使用HTML, JavaScript, CSS实现, 带有少量PHP。

系统入口部分为HTML+CSS搭建的静态页面, JavaScript实现表单数据格式校验并向后端传输数据。

系统主页面主要为JavaScript+CSS动态渲染页面，因此能够实现面向不同用户显示不同页面，以及在同一个php页面显示所有功能板块。

系统前后段交互主要采用JavaScript向后端发送Ajax请求。

后端

系统后端主要使用PHP实现。

通过对Ajax请求附带的信息进行判断来判断要进行的操作。
